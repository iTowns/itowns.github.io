Navigation extends Widget

A widget menu manager for navigation.

To use it, you need to link the widgets' stylesheet to your html webpage. This stylesheet is included in itowns bundles if you downloaded them, or it can be found in node_modules/itowns/examples/css if you installed iTowns with npm. Otherwise, it can be found at this link. See this example for more details.

Constructor

Navigation ( view: GlobeView, options: Object )

Parameters

  • view —

    The iTowns view the navigation should be linked to. For the moment, only GlobeView is supported.
  • options —

    The navigation menu optional configuration.
    • options.parentElement (default: view.domElement) —

      The parent HTML container of the div which contains navigation widgets.
    • options.displayCompass (default: true) —

      Whether the compass widget should be displayed.
    • options.display3DToggle (default: true) —

      Whether the navigation should include a widget to toggle between top and oblique view.
    • options.displayZoomIn (default: true) —

      Whether the zoom-in widget should be displayed.
    • options.displayZoomOut (default: true) —

      Whether the zoom-out widget should be displayed.
    • options.animationDuration (default: 500) —

      The duration of travel animations, when clicking navigation widgets.
    • options.position (default: 'bottom-left') —

      Defines which corner of the parentElement the navigation menu should be displayed to. Possible values are top-left, top-right, bottom-left and bottom-right. If the input value does not match one of these, it will be defaulted to bottom-left.
    • options.direction (default: 'column') —

      Whether the navigation menu should expand horizontally or vertically. Possible values are column and row. If the input value does not match one of these, it will be defaulted to column.
    • options.translate (optional) —

      An optional translation of the navigation menu.
      • options.translate.x (optional) —

        The navigation menu translation along the page x-axis.
      • options.translate.y (optional) —

        The navigation menu translation along the page y-axis.

Example

// Create a Navigation widget in the bottom-right corner of an iTowns view domElement
const navigation = new Navigation(view, { position: 'bottom-right' };

// Change the tooltip for the compass :
navigation.compass.title = 'new tooltip';

// Change the method ran when clicking zoom-in button :
function newMethod() {
    // do something
}
navigation.zoomIn.onclick = newMethod;

Properties

domElement : HTMLElement

An html div containing all navigation widgets.

parentElement : HTMLElement

The parent HTML container of this.domElement.

compass : HTMLButtonElement

The HTML button for the compass.

toggle3D : HTMLButtonElement

The HTML button for the 3D/2D toggle button.

zoomIn : HTMLButtonElement

The HTML button for the zoom-in button.

zoomOut : HTMLButtonElement

The HTML button for the zoom-out button.

Methods

addButton ( id: string, content: string, title: string, onclick: function, parentId: string ) : HTMLButtonElement

Returns

The created button.

Parameters

  • id —

    The unique id the created button should be given.
  • content (default: '') —

    An HTML string defining the content of the button.
  • title (default: '') —

    An HTML string defining information on the button. This string will be displayed in a tooltip when hovering the button.
  • onclick (optional) —

    The method that should be executed when the button is clicked on.
  • parentId (optional) —

    The unique id of a button bar in which the created button should be added. A button bar is a group which contains one or several buttons. All buttons created with Navigation are in a button bar. If the given id does not match an already existing button bar, a new button bar will be created with this id. If no id is given, a button bar will be created with no id. The later case can be useful for creating isolated buttons.

hide ( )

Change the widget style display property so that the widget becomes invisible.

show ( )

Change the widget style display property so that the widget becomes visible.
