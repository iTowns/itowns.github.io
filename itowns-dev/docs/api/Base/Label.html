Label

An object that handles the display of a text and/or an icon, linked to a 3D position. The content of the Label is managed through a DOM object, in a <div> handled by the Label2DRenderer.

Constructor

Label ( content: Element|string, coordinates: Coordinates, style: Style, sprites: Object )

Parameters

  • content —

    The content; can be a string, with or without HTML tags in it, or it can be an Element.
  • coordinates —

    The world coordinates, where to place the Label.
  • style —

    The style to apply to the content. Once the style is applied, it cannot be changed directly. However, if it really needed, it can be accessed through label.content.style, but it is highly discouraged to do so.
  • sprites (optional) —

    the sprites.

Properties

isLabel : boolean

Used to checkout whether this object is a Label. Default is true. You should not change this, as it is used internally for optimisation.

content : Element

The DOM object that contains the content of the label. The style and the position are applied on this object. All labels contain the itowns-label class, as well as a specific class related to the layer linked to it: itowns-label-[layer-id] (replace [layer-id] by the correct string).

position : THREE.Vector3

The position in the 3D world of the label.

padding : number

sets the padding area on all four sides of an element at once.

coordinates : Coordinates

The coordinates of the label.

order : number

Order of the label that will be read from the style. It helps sorting and prioritizing a Label during render.

Methods

initDimensions ( )

Updates the screen dimensions of the label, using getBoundingClientRect. It updates width and height property of the label, and the boundaries.

updateProjectedPosition ( x: number, y: number )

Moves a label on the screen, using screen coordinates. It updates the boundaries as it moves it.

Parameters

  • x —

    X coordinates in pixels, from left.
  • y —

    Y coordinates in pixels, from top.
