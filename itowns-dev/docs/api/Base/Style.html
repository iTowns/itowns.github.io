Style

A Style is an object that defines the visual appearance of FeatureCollection and Feature. It is taken into account when drawing them in textures that will be placed onto tiles.

As there are four basic elements present in Features, there are also four main components in a Style object:

  • fill is for all fillings and polygons
  • stroke is for all lines and polygons edges
  • point is for all points
  • text contains all Label related things

Many style property can be set to functions. When that is the case, the function's return type must necessarily be the same as the types (other than function) of the property. For instance, if the fill.pattern property is set to a function, this function must return an Image, a Canvas, or String. The first parameter of functions used to set Style properties is always an object containing the properties of the features displayed with the current Style instance.

Constructor

Style ( params: Object, parent: Style )

Parameters

  • params (default: {}) —

    An object that can contain all properties of a Style.
  • parent (optional) —

    The parent style, that is looked onto if a value is missing.

Example

const style = new itowns.Style({
     stroke: { color: 'red' },
     point: { color: 'white', line: 'red' },
});

const source = new itowns.FileSource(...);

const layer = new itowns.ColorLayer('foo', {
     source: source,
     style: style,
});

view.addLayer(layer);

Properties

order : number

Order of the features that will be associated to the style. It can helps sorting and prioritizing features if needed.

fill : Object

Polygons and fillings style.

fill.color : string|function

Defines the main color of the filling. Can be any valid color string. Default is no value, indicating that no filling needs to be done. If the Layer is a GeometryLayer you could use THREE.Color.

fill.pattern : Image|Canvas|string|function

Defines a pattern to fill the surface with. It can be an Image to use directly, or an url to fetch the pattern from. See this example for how to use.

fill.opacity : number|function

The opacity of the color or the pattern. Can be between 0.0 and 1.0. Default is 1.0. For a GeometryLayer, this opacity property isn't used.

fill.base_altitude : number|function

GeometryLayer Style, defines altitude for each Coordinates. If base_altitude is undefined, the original altitude is kept, and if it doesn't exist then the altitude value is set to 0.

fill.extrusion_height : number|function

GeometryLayer Style, if defined, polygons will be extruded by the specified amount

stroke : Object

Lines and polygon edges.

stroke.color : string|function

The color of the line. Can be any valid color string. Default is no value, indicating that no stroke needs to be done. If the Layer is a GeometryLayer you could use THREE.Color.

stroke.opacity : number|function

The opacity of the line. Can be between 0.0 and 1.0. Default is 1.0. For a GeometryLayer, this opacity property isn't used.

stroke.width : number|function

The width of the line. Default is 1.0.

stroke.base_altitude : number|function

GeometryLayer Style, defines altitude for each Coordinates. If base_altitude is undefined, the original altitude is kept, and if it doesn't exist then the altitude value is set to 0.

point : Object

Point style.

point.color : string|function

The color of the point. Can be any valid color string. Default is no value, indicating that no point will be shown.

point.radius : number|function

The radius of the point, in pixel. Default is 2.0.

point.line : string|function

The color of the border of the point. Can be any valid color string. It's not supported with a GeometryLayer.

point.width : number|function

The width of the border, in pixel. Default is 0.0 (no border).

point.opacity : number|function

The opacity of the point. Can be between 0.0 and 1.0. Default is 1.0. It's not supported with a GeometryLayer.

point.base_altitude : number|function

GeometryLayer Style, defines altitude for each Coordinates. If base_altitude is undefined, the original altitude is kept, and if it doesn't exist then the altitude value is set to 0.

text : Object

All things Label related.

text.field : string|function

A string to help read the text field from properties of a FeatureGeometry, with the key of the property enclosed by brackets. For example, if each geometry contains a name property, text.field can be set as {name}. Default is no value, indicating that no text will be shown.

The brackets allows the use of complex fields. For example, if a static string foo is attached to the changing property bar, you can specify foo {bar}, and foo will stay everytime, while {bar} will change into the value of the property of the geometry. You can also have multiple properties in one field, like if you want the latin name and the local name of a location. Specifying {name_latin} - {name_local} can result in Marrakesh - مراكش for example.

text.color : string|function

The color of the text. Can be any valid color string. Default is #000000.

text.anchor : string|Array.<number>|function

The anchor of the text compared to its position (see Label for the position). Can be a few value: top, left, bottom, right, center, top-left, top-right, bottom-left or bottom-right. Default is center.

It can also be defined as an Array of two numbers. Each number defines an offset (in fraction of the label width and height) between the label position and the top-left corner of the text. The first value is the horizontal offset, and the second is the vertical offset. For example, [-0.5, -0.5] will be equivalent to center.

text.offset : Array|function

The offset of the text, depending on its anchor, in pixels. First value is from left, second is from top. Default is [0, 0].

text.padding : number|function

The padding outside the text, in pixels. Default is 2.

text.size : number|function

The size of the font, in pixels. Default is 16.

text.wrap : number|function

The maximum width, in pixels, before the text is wrapped, because the string is too long. Default is 10.

text.spacing : number|function

The spacing between the letters, in em. Default is 0.

text.transform : string|function

A value corresponding to the CSS property text-transform. Default is none.

text.justify : string|function

A value corresponding to the CSS property text-align. Default is center.

text.opacity : number|function

The opacity of the text. Can be between 0.0 and 1.0. Default is 1.0.

text.font : Array|function

A list (as an array of string) of font family names, prioritized in the order it is set. Default is Open Sans Regular, Arial Unicode MS Regular, sans-serif.

text.haloColor : string|function

The color of the halo. Can be any valid color string. Default is #000000.

text.haloWidth : number|function

The width of the halo, in pixels. Default is 0.

text.haloBlur : number|function

The blur value of the halo, in pixels. Default is 0.

icon : Object

Defines the appearance of icons attached to label.

icon.source : String

The url of the icons' image file.

icon.key : String

The key of the icons' image in a vector tile data set.

icon.anchor : string

The anchor of the icon compared to the label position. Can be left, bottom, right, center, top-left, top-right, bottom-left or bottom-right.

icon.size : number

If the icon's image is passed with icon.source or icon.key, it size when displayed on screen is multiplied by icon.size. Default is 1.
