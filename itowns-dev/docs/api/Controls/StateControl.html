StateControl

It represents the control's states. Each State is a control mode of the camera and how to interact with the interface to activate this mode.

Constructor

StateControl ( )

Properties

NONE : State

State when camera is idle.

ORBIT : State

State describing camera orbiting movement : the camera moves around its target at a constant distance from it.

DOLLY : State

State describing camera dolly movement : the camera moves forward or backward from its target.

PAN : State

State describing camera pan movement : the camera moves parallel to the current view plane.

MOVE_GLOBE : State

State describing camera drag movement : the camera is moved around the view to give the feeling that the view is dragged under a static camera.

PANORAMIC : State

State describing camera panoramic movement : the camera is rotated around its own position.

TRAVEL_IN : State

State describing camera travel in movement : the camera is zoomed in toward a given position. The target position depends on the key/mouse binding of this state. If bound to a mouse button, the target position is the mouse position. Otherwise, it is the center of the screen.

TRAVEL_OUT : State

State describing camera travel out movement : the camera is zoomed out from a given position. The target position depends on the key/mouse binding of this state. If bound to a mouse button, the target position is the mouse position. Otherwise, it is the center of the screen. It is disabled by default.

ZOOM : State

State describing camera zoom in and out movement.

enable : boolean

Defines whether all input will be communicated to the associated Controls or not. Default is true.

enableKeys : boolean

Defines whether keyboard input will be communicated to the associated Controls or not. Default is true.

Methods

dispose ( )

Remove all event listeners created within this instance of StateControl

inputToState ( mouseButton: Number, keyboard: Number, double: Boolean ) : State

get the state corresponding to the mouse button and the keyboard key. If the input relates to a trigger - a single event which triggers movement, without the move of the mouse for instance -, dispatch a relevant event.

Returns

the state corresponding

Parameters

  • mouseButton —

    The mouse button
  • keyboard —

    The keyboard
  • double (optional) —

    Value of the searched state double property

setFromOptions ( options: Object )

Set the current StateControl State properties to given values.

Parameters

  • options —

    Object containing the State values to set current StateControl properties to. The enable property do not necessarily need to be specified. In that case, the previous value of this property will be kept for the new State.

Example

// Switch bindings for PAN and MOVE_GLOBE actions, and disabling PANORAMIC movement :
view.controls.states.setFromOptions({
    PAN: {
        mouseButton: itowns.THREE.MOUSE.LEFT,
    },
    MOVE_GLOBE: {
        mouseButton: itowns.THREE.MOUSE.RIGHT,
    },
    PANORAMIC: {
        enable: false,
    },
};

Returns

the state corresponding

Parameters

  • finger —

Types

State : Object

  • enable: boolean (default: true) —

    Indicate whether the state is enabled or not.
  • mouseButton: Number (optional) —

    The mouse button bound to this state.
  • keyboard: Number (optional) —

    The keyCode of the keyboard input bound to this state.
  • finger: Number (optional) —

    The number of fingers on the pad bound to this state.
  • double: boolean (optional) —

    True if the mouse button bound to this state must be pressed twice. For example, if double is set to true with a mouseButton set to left click, the State will be bound to a double click mouse button.
