LASParser

The LASParser module provides a parse method that takes a LAS or LAZ (LASZip) file in, and gives a THREE.BufferGeometry containing all the necessary attributes to be displayed in iTowns. It uses the copc.js library.

Static Methods

parse ( data: ArrayBuffer, options: Object ) : Promise

Parses a LAS file or a LAZ (LASZip) file and return the corresponding THREE.BufferGeometry.

Returns

A promise resolving with a THREE.BufferGeometry. The header of the file is contained in userData.

Parameters

  • data —

    The file content to parse.
  • options (optional)
    • options.in (optional) —

      Options to give to the parser.
      • options.in.colorDepth (optional) —

        Color depth (in bits). Defaults to 8 bits for LAS 1.2 and 16 bits for later versions (as mandatory by the specification)

parseChunk ( data: ArrayBuffer, options: Object ) : Promise.<THREE.BufferGeometry>

Parses a chunk of a LAS or LAZ (LASZip) and returns the corresponding THREE.BufferGeometry.

Returns

A promise resolving with a THREE.BufferGeometry.

Parameters

  • data —

    The file content to parse.
  • options
    • options.in —

      Options to give to the parser.
      • options.in.pointCount —

        Number of points encoded in this data chunk.
      • options.in.header —

        Partial LAS file header.
        • options.in.header.pointDataRecordFormat —

          Type of Point Data Record contained in the LAS file.
        • options.in.header.pointDataRecordLength —

          Size (in bytes) of the Point Data Record.
    • options.eb (optional) —

      Extra bytes LAS VLRs headers.
    • options.in.colorDepth (optional) —

      Color depth (in bits). Defaults to 8 bits for LAS 1.2 and 16 bits for later versions (as mandatory by the specification)

terminate ( ) : Promise.<void>

Terminate all worker instances.
