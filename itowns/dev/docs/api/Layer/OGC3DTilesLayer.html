OGC3DTilesLayer extends Layer

Layer for 3D Tiles datasets.

Advanced configuration note: 3D Tiles rendering is delegated to 3DTilesRendererJS that exposes several configuration options accessible through the tilesRenderer property of this class. see the 3DTilesRendererJS doc. Also note that the cache is shared amongst 3D tiles layers and can be configured through tilesRenderer.lruCache (see the following documentation.

Constructor

OGC3DTilesLayer ( id: String, config: Object )

Parameters

  • id —

    unique layer id.
  • config —

    layer specific configuration
    • config.source —

      data source configuration
    • config.pntsMode (default: PNTS_MODE.COLOR) —

      Point cloud coloring mode (passed to PointsMaterial). Only 'COLOR' or 'CLASSIFICATION' are possible. COLOR uses RGB colors of the points, CLASSIFICATION uses a classification property of the batch table to color points.
    • config.classificationScheme (default: ClassificationScheme.DEFAULT) —

      PointsMaterial classification scheme
    • config.pntsShape (default: PNTS_SHAPE.CIRCLE) —

      Point cloud point shape. Only 'CIRCLE' or 'SQUARE' are possible. (passed to PointsMaterial).
    • config.pntsSizeMode (default: PNTS_SIZE_MODE.VALUE) —

      PointsMaterial Point cloud size mode (passed to PointsMaterial). Only 'VALUE' or 'ATTENUATED' are possible. VALUE use constant size, ATTENUATED compute size depending on distance from point to camera.
    • config.pntsMinAttenuatedSize (default: 3) —

      Minimum scale used by 'ATTENUATED' size mode.
    • config.pntsMaxAttenuatedSize (default: 10) —

      Maximum scale used by 'ATTENUATED' size mode.

Methods

defineLayerProperty ( propertyName: string, defaultValue: *, onChange: function )

Defines a property for this layer, with a default value and a callback executed when the property changes.

When changing the property, it also emits an event, named following this convention: ${propertyName}-property-changed, with ${propertyName} being replaced by the name of the property. For example, if the added property name is frozen, it will emit a frozen-property-changed.

Parameters

  • propertyName —

    The name of the property, also used in the emitted event name.
  • defaultValue —

    The default set value.
  • onChange (optional) —

    The function executed when the property is changed. Parameters are the layer the property is defined on, and the name of the property.

Example

The emitted event has some properties assigned to it 
event = {
    new: {
        ${propertyName}: * // the new value of the property
    },
    previous: {
        ${propertyName}: * // the previous value of the property
    },
    target: Layer // the layer it has been dispatched from
    type: string // the name of the emitted event
}

delete ( )

Deletes the layer and frees associated memory

forEachTile ( callback: function )

Executes a callback for each tile of this layer tileset.

Parameters

  • callback —

    The callback to execute for each tile. Has the following two parameters:

    1. tile (Object) - the JSON tile
    2. scene (THREE.Object3D | null) - The tile content. Contains a batchTable property. Can be null if the tile has not yet been loaded.

Fired when a tile model is loaded

Properties

Fired when a new root or child tile set is loaded

Properties

tile-visibility-change

Fired when a tiles visibility changes

Properties

    the model (tile content) parsed in a THREE.GROUP
  • tile: Object —

    the tile metadata from the tileset

Fired when all visible tiles are loaded (can be fired multiple times, e.g. when the camera moves and new tiles are loaded)

Fired when a new batch of tiles start loading (can be fired multiple times, e.g. when the camera moves and new tiles start loading)
