ColorLayer extends RasterLayer

A simple layer, usually managing a texture to display on a view. For example, it can be an aerial view of the ground or a simple transparent layer with the roads displayed.

Constructor

ColorLayer ( id: string, config: Object )

Parameters

  • id —

    The id of the layer, that should be unique. It is not mandatory, but an error will be emitted if this layer is added a View that already has a layer going by that id.
  • config (optional) —

    Optional configuration, all elements in it will be merged as is in the layer. For example, if the configuration contains three elements name, protocol, extent, these elements will be available using layer.name or something else depending on the property name.
    • config.source (optional) —

      Description and options of the source.
    • config.magFilter (optional) —

      How the texture is sampled when a texel covers more than one pixel. see
    • config.minFilter (optional) —

      How the texture is sampled when a texel covers less than one pixel. see
    • config.effect_type (optional) —

      type effect to apply on raster color. if effect_type equals:

      • 0: no special effect.
      • 1: light color to invisible effect.
      • 2: white color to invisible effect.
      • 3: custom shader effect (defined ShaderChunk.customBodyColorLayer and ShaderChunk.customHeaderColorLayer).
    • config.effect_parameter (default: 1.0) —

      amount value used with effect applied on raster color.

Example

// Create a ColorLayer
const color = new ColorLayer('roads', {
    source: new  SourceWMTS({
         protocol: 'wmts',
         url: 'http://server.geo/wmts/....',
         format: 'image/png',
         name: 'nameService',
         tileMatrixSet: 'PM',
    }),
    transparent: true,
    opacity: 0.5,
});

// Add the layer
view.addLayer(color);

Properties

isColorLayer : boolean

Used to checkout whether this layer is a ColorLayer. Default is true. You should not change this, as it is used internally for optimisation.

style : StyleOptions|Style

style properties or a Style defined by the user, to apply to the layer features.

visible : boolean

property to display or to hide layer.

opacity : number

property to adjust transparency, opacity is between 0. and 1.

transparent : boolean

specify if the layer could be transparent.

noTextureParentOutsideLimit : boolean

don't parent texture if it's outside limit.

effect_type : number

type effect to apply on the raster color. If effect_type equals:

  • 0: no effect.
  • 1: light color to invisible effect: transparency effect is stronger for light colors (whose distance is closer to white). This effect can be amplified by effect_parameter.
  • 2: white color to invisible effect: white color is considered as transparent.
  • 3: custom shader effect: set ShaderChunk.customBodyColorLayer and ShaderChunk.customHeaderColorLayer to add your own glsl code in respectively the color layer function body and at top-level.

effect_parameter : number

value used as parameter of certain type of effects. If effect_type equals:

  • 0: unused.
  • 1: used to amplify the transparency effect.
  • 2: unused.
  • 3: could be used by your own glsl code.

Methods

delete ( clearCache: boolean )

All layer's textures are removed from scene and disposed from video device.

Parameters

  • clearCache (optional) —

    Whether to clear the layer cache or not

setupRasterNode ( node: TileMesh ) : RasterColorTile

Setup RasterColorTile added to TileMesh. This RasterColorTile handles the ColorLayer textures mapped on this TileMesh.

Returns

The raster color node added.

Parameters

  • node —

    The node to apply new RasterColorTile;

Events

opacity-property-changed

Fires when the opacity of the layer has changed.

sequence-property-changed

Fires when the sequence of the layer has changed, meaning that the order of the layer changed in the view it is attached to.

visible-property-changed

Fires when the visiblity of the layer has changed.
